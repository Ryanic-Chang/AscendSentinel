# AscendSentinel

AscendSentinel 是一个基于昇腾 AIpro 平台的大模型舆情监控原型系统，面向真实舆情分析场景，完成了从模型部署、推理接口统一到文本与多模态舆情理解的端到端流程验证。项目以 Qwen 系列模型为核心，结合轻量级交互界面，探索大模型在实际舆情监测任务中的工程化落地方式。

该项目的目标并非构建复杂的业务系统，而是验证在昇腾 AIpro 环境下，大模型在文本理解、多任务分析以及后续多模态扩展中的可行性、稳定性与可扩展性，为后续平台化或产品化迭代提供基础。

## 项目背景

随着大模型能力不断增强，舆情分析正从传统规则与分类模型逐步转向以 LLM 为核心的统一理解范式。本项目围绕“如何在国产算力平台上高效部署并使用大模型完成舆情分析任务”这一问题展开，选用昇腾 AIpro 8T16G 环境与 MindSpore 生态，对 Qwen1.5-0.5B 模型进行适配与部署，并构建统一的推理调用方式，以支持后续不同分析任务的复用。

在此基础上，系统实现了对用户评论、话题文本的解析与理解，并预留了多模态舆情分析的扩展能力，用于探索文本与图像联合分析的可能性。

## 功能概述

系统围绕舆情分析的核心需求，完成了文本层面的情感倾向分析、用户意图识别与主题聚合能力。通过对输入文本进行统一预处理与分发，模型能够在一次推理流程中完成多维度分析结果的生成，减少多模型串联带来的复杂性与性能开销。

在此基础上，项目进一步集成了 Qwen-VL 相关能力，验证了图像角色与语义信息在舆情理解场景中的补充价值，为后续复杂事件分析提供支持。

## 工作流说明

项目整体工作流以 `monitor.py` 作为核心入口，负责模型加载、推理调度与结果整合。系统启动后，程序会读取输入文本数据，对内容进行标准化处理，并通过统一的推理接口调用已部署的大模型完成分析任务。模型返回的结果会被解析为结构化信息，用于后续展示或进一步处理。

当前仓库中提供的 `topic.txt` 与 `user.txt` 作为示例测试用例，分别用于模拟舆情话题文本与用户评论输入，方便在不同环境下快速验证模型推理与分析逻辑的正确性。

## 目录结构说明

当前仓库结构保持简洁，核心逻辑集中在 `monitor.py` 文件中，用于演示完整的舆情分析流程。测试用例以纯文本形式提供，便于直接替换或扩展为真实数据源。项目采用 MIT License，便于后续学习、修改与二次开发。

## 运行说明

在完成昇腾 AIpro 环境配置及相关依赖安装后，可直接运行 `monitor.py` 启动舆情分析流程。通过修改或替换测试用例文本，可快速验证模型在不同输入条件下的分析效果。项目当前以功能验证与流程演示为主，未对性能与并发进行专项优化。

## 说明与声明

本项目为研究与工程实践性质的原型系统，主要用于探索大模型在舆情分析场景中的部署方式与应用形式，不构成实际业务系统或商用产品。模型能力与分析结果依赖于所使用的大模型版本及推理配置，结果仅供参考。
